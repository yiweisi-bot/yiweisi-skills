- **名称**: task-tracker
- **作者**: rikouu
- **功能**: 主动式任务状态管理
- **核心理念**: 维护一个实时任务状态文件，让上下文在会话重置/压缩后仍然能够继续

### agent-task-tracker 详细功能

#### 状态文件
- **位置**: `memory/tasks.md`
- **作用**: 单一事实来源（single source of truth）

#### 写入时机
1. **收到任务** → 添加条目，状态设为 `🔄 进行中`
2. **后台进程启动** → 记录 session ID、PID、服务器、命令
3. **进度更新** → 更新状态/备注
4. **任务完成** → 标记 `✅ 完成`，记录结果/链接
5. **任务失败** → 标记 `❌ 失败`，记录错误
6. **会话开始** → 读取 `memory/tasks.md` 恢复意识

#### 文件格式
```markdown
# Active Tasks

## [task-id] 简短描述
- **Status**: 🔄 进行中 | ✅ 完成 | ❌ 失败 | ⏸️ 暂停
- **Requested**: YYYY-MM-DD HH:MM
- **Updated**: YYYY-MM-DD HH:MM
- **Background**: session-id (PID) on server-name — `command`
- **Notes**: 进度详情、部分结果
- **Result**: 最终输出、链接、总结

# Completed (recent)
<!-- 把完成的任务移到这里，保留最近10个，删除旧的 -->
```

#### 核心规则
1. **先写后报** - 在向用户报告之前先更新文件
2. **足够详细** - 包含足够细节以便在没有先前对话上下文的情况下恢复
3. **后台进程** - 总是记录 session ID + 服务器 + 命令
4. **多步骤任务** - 每步后更新
5. **保持简洁** - 这不是日志，是状态快照
6. **大小限制** - 保持在 50 行 / 2KB 以内
7. **完成任务** - 压缩成一行摘要，详细信息参考每日笔记
8. **清理旧任务** - 删除超过 3 天的完成任务
9. **空状态提示** - 如果没有活动任务，写（无）让它明显

#### 价值
解决了 OpenClaw 的一个痛点：**会话重置后上下文丢失**。通过维护一个独立的 `memory/tasks.md` 文件，即使会话被压缩或重置，任务状态也能保留下来。
