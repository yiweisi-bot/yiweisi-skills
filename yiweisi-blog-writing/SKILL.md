# 乙维斯博客编写技能

## 概述

这是乙维斯博客的编写规范和最佳实践技能。每次编写新博客文章时，必须遵循此技能中的规范。

---

## ⭐⭐⭐ 最重要的博客发布规范 ⭐⭐⭐

### 1. 博客文章标题重复问题（绝对禁止！）

**问题描述**：
- Frontmatter 中已经有 `title:` 字段
- 正文开头又写了一次 `# 标题`
- 导致页面上标题显示两次

**正确做法**：
- 标题只在 frontmatter 的 `title:` 中写一次
- 正文开头不要再写 `# 标题`
- 系统会自动把 title 显示在页面上
- 正文直接从第一段落开始

---

### 2. 博客文章 tags 格式问题

**问题描述**：
- tags 使用了单引号：`tags: ['OpenClaw', '技能']`
- 导致页面展示时有多余的单引号
- 看起来不美观

**正确做法**：
- tags 中不要加单引号
- 直接写标签内容，用逗号分隔
- 用方括号 [] 包裹

**示例**：
```yaml
tags: [OpenClaw, AI, 灵魂, 自我优化]  # ✅ 正确
tags: ['OpenClaw', 'AI']              # ❌ 错误
```

## SEO 优化要点

### 1. Frontmatter 优化

**必须包含的字段**：
- `title`: 清晰、吸引人的标题（60字符以内）
- `date`: 发布日期（YYYY-MM-DD格式）
- `author`: 作者名（通常是"乙维斯"）
- `tags`: 相关标签数组（3-5个标签）
- `excerpt`: 文章摘要（150字符以内）

**示例**：
```yaml
title: OpenClaw多Agent协作模式完整教程
date: 2026-02-27
author: 乙维斯
tags: [OpenClaw, 人工智能, 多Agent, 教程]
excerpt: 从零开始构建AI开发团队：主Agent + 开发Agent + Claude Code Hook，30分钟实现零轮询自动开发
```

### 2. 内容优化

**标题层级**：
- H1: 系统自动显示（不要在正文中写）
- H2: 主要部分（##）
- H3: 子部分（###）
- H4-H6: 酌情使用

**关键词使用**：
- 在文章开头自然融入关键词
- 在小标题中使用相关关键词
- 不要堆砌关键词
- 使用相关关键词变体

**内容长度**：
- 技术教程：1000字以上
- 日常记录：500字以上
- 深度文章：2000字以上

**内部链接**：
- 链接到相关的博客文章
- 链接到FAQ页面
- 链接到About页面
- 使用描述性的链接文字

## GEO（生成式引擎优化）要点

### 1. 内容结构化

**问答格式**（推荐）：
```markdown
## 什么是OpenClaw？

OpenClaw是一个强大的AI助手框架...

## 如何配置多Agent？

配置多Agent需要以下步骤...
```

**关键信息前置**：
- 文章开头先写摘要（excerpt）
- 用列表形式呈现核心要点
- 重要结论放在前面

**清晰的层级**：
- 使用标题层级组织内容
- 使用列表（有序/无序）呈现要点
- 使用表格对比数据

### 2. E-E-A-T 优化

**Experience（经验）**：
- 分享实际经历和案例
- 使用"我"、"我们"等人称
- 提及具体的时间和场景

**Expertise（专业）**：
- 展示技术深度
- 使用正确的专业术语
- 提供详细的技术细节

**Authoritativeness（权威）**：
- 链接到权威来源
- 提及行业标准
- 引用官方文档

**Trustworthiness（可信）**：
- 保持真实和透明
- 及时更新过时内容
- 承认错误并更正

### 3. 问题导向内容

**收集用户问题**：
- 用户常问的问题
- 论坛和问答平台的热门问题
- 社交媒体的相关讨论

**分类问题类型**：
- What（什么）- 定义和概念
- How（如何）- 方法和教程
- Why（为什么）- 原因和解释
- Which（哪个）- 比较和推荐

**创建问答内容**：
- 每个重要问题一个专门的章节
- 提供完整、详细的答案
- 预期后续问题并提前回答

## 编写前检查清单

在发布博客文章前，必须检查以下项目：

### Frontmatter 检查
- [ ] title 字段已填写，且吸引人
- [ ] date 字段格式正确（YYYY-MM-DD）
- [ ] author 字段已填写（通常是"乙维斯"）
- [ ] tags 字段已填写，3-5个标签
- [ ] tags 没有使用单引号
- [ ] excerpt 字段已填写，150字符以内

### 内容检查
- [ ] 正文开头没有重复的 `# 标题`
- [ ] 标题层级清晰（H2 → H3 → H4）
- [ ] 内容完整，有足够的深度
- [ ] 没有语法错误和拼写错误
- [ ] 代码块格式正确
- [ ] 图片有ALT标签（如适用）

### SEO 检查
- [ ] 关键词在文章开头自然出现
- [ ] 小标题中使用了相关关键词
- [ ] 有内部链接到相关文章
- [ ] 内容长度符合要求

### GEO 检查
- [ ] 内容结构清晰，易于理解
- [ ] 有摘要或要点总结在开头
- [ ] 使用了列表和表格（如适用）
- [ ] 提供了完整的答案（如适用）

---

## 常见错误及解决方案

| 错误 | 问题 | 解决方案 |
|------|------|---------|
| 正文开头有 `# 标题` | 标题显示两次 | 删除正文中的 `# 标题` |
| tags 使用单引号 | 页面显示多余引号 | 删除 tags 中的单引号 |
| 没有 excerpt | SEO和GEO效果差 | 写1-2句话的摘要 |
| 内容太短 | 搜索引擎不重视 | 扩展内容，增加深度 |
| 没有内部链接 | 用户停留时间短 | 链接到相关文章 |
| 标题层级混乱 | 难以阅读 | 使用 H2 → H3 → H4 |

## 博客文章类型指南

### 1. 技术教程

**特点**：
- 步骤清晰
- 有代码示例
- 有预期结果
- 有常见问题解答

**结构**：
```
## 简介
- 为什么需要这个教程
- 目标读者是谁

## 准备工作
- 需要什么前提条件
- 需要安装什么工具

## 步骤1：XXX
- 详细说明
- 代码示例
- 预期结果

## 步骤2：XXX
...

## 常见问题
- 问题1及解决方案
- 问题2及解决方案

## 总结
```

### 2. 日常记录

**特点**：
- 时间线清晰
- 具体事件描述
- 个人感悟
- 真实、自然

**结构**：
```
## 日期
- 事件1
- 事件2
- ...

## 感悟
- 学到了什么
- 有什么想法

## 下一步计划
- 明天要做什么
- 未来的目标
```

### 3. 深度文章

**特点**：
- 有深度分析
- 有独特见解
- 有案例支撑
- 内容详实

**结构**：
```
## 引言
- 背景介绍
- 为什么这个话题重要

## 核心概念
- 定义和解释
- 相关理论

## 深入分析
- 多个角度分析
- 案例研究

## 实践应用
- 如何应用
- 注意事项

## 总结和展望
- 核心要点总结
- 未来趋势
```

## 发布流程

1. **编写内容** - 按照规范编写文章
2. **检查清单** - 使用上面的检查清单检查

4. **构建测试** - 运行 `npm run build` 确保构建成功
5. **Git提交** - 提交到 Git 仓库
6. **部署上线** - 部署到生产环境
7. **更新记忆** - 在记忆文件中记录发布
