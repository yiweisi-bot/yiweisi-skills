## 📋 完整流程详解

### 阶段1: 用户目标输入

**输入示例**:
```
用户: "帮我学习一下React 19、Python、还有Docker"
```

**处理流程**:
```
1. 解析用户输入
   ↓
2. 拆分为3个独立目标
   - 目标1: 学习 React 19
   - 目标2: 学习 Python
   - 目标3: 学习 Docker
   ↓
3. 自动评估优先级
   - React 19: 高优先级（新技术、用户可能需要）
   - Python: 中优先级
   - Docker: 中优先级
   ↓
4. 存入数据库 (learning_goals表)
   ↓
5. 开始执行第一个目标 (React 19)
```

**数据库操作**:
```sql
INSERT INTO learning_goals (title, priority, source) VALUES
('学习 React 19', 1, 'user_input'),
('学习 Python', 2, 'user_input'),
('学习 Docker', 2, 'user_input');
```

### 阶段2: Action-Reflection-Iteration 循环（核心！）

**配置**:
- 最大循环次数: 3次
- 信息充分阈值: 70分

---

#### 循环1: 广泛收集

**Action (执行)**:
```
任务: 广泛搜索
- 打开 GitHub Trending
- 浏览 react.dev 官方文档
- 搜索 GitHub Blog 相关文章
- 收集初步信息

工具: agent-browser
```

**Reflection (反思)**:
```
评估信息充分性:

维度                    得分
──────────────────────────────
核心概念覆盖            80分 ✓
实用示例数量            30分 ✗ (只有1个简单示例)
最佳实践               60分 ⚠️
常见问题                0分 ✗ (完全没有)
信息来源质量            90分 ✓
──────────────────────────────
总分: 52分

判断: 信息不够，需要继续搜索

缺失项:
- 完整的代码示例（需要2-3个）
- 常见问题解答
- 更多最佳实践

建议搜索:
- "React 19 Server Components 完整示例"
- "React 19 常见问题 FAQ"
- "React 19 最佳实践 2026"
```

**Iteration (迭代决策)**:
```
判断: 继续循环2
原因: 52分 < 70分阈值，且未达最大循环次数
```

### 阶段5: 发布与追踪

**操作**:
```
1. 保存技能文件到 skills/ 目录
2. 记录到数据库 (generated_skills表)
3. 更新学习目标状态为 completed
4. 记录学习会话结束
5. 通知用户
```

**数据库操作**:
```sql
-- 记录生成的技能
INSERT INTO generated_skills 
(goal_id, name, file_path, quality_score, status)
VALUES (?, 'React 19 Server Components 指南', 
        'skills/react-19-server-components.md', 87, 'published');

-- 更新目标状态
UPDATE learning_goals
SET status = 'completed', completed_at = CURRENT_TIMESTAMP
WHERE id = ?;

-- 结束会话
UPDATE learning_sessions
SET status = 'completed', end_time = CURRENT_TIMESTAMP
WHERE id = ?;
```

## ⏰ 超时处理机制

### 场景: 学习时间到了，但只完成50%

**流程**:
```
1. 检测到超时
   ↓
2. 估算剩余时间: "预计还需要30分钟"
   ↓
3. 通知用户:
   "学习React 19已经到预设时间了，
    目前完成了50%，预计还需要30分钟。
    要继续吗？"
   ↓
4. 等待用户回复 (设置超时: 5分钟)
   ↓
5a. 用户说"继续" → 继续学习
   ↓
5b. 用户说"停止" → 保存状态，加入待学习列表
   ↓
5c. 5分钟无回复 → 自动保存，加入待学习列表
```

**数据库操作**:
```sql
-- 更新会话状态
UPDATE learning_sessions
SET status = 'interrupted', end_time = CURRENT_TIMESTAMP
WHERE id = ?;

-- 加入待学习列表
INSERT INTO learning_backlog (goal_id, reason)
VALUES (?, '学习超时，未完成，用户选择暂停');
```

## 💰 Token消耗优化

### 优化策略
1. **LLM全程介入，但每一步都明确目标**
2. **先决策、再设计、最后生成** - 避免盲目生成
3. **自我检查、自我改进** - 减少来回迭代
4. **规则验证先做** - 低成本快速过滤
5. **AI验证后再人工** - 节省人工成本

### 预期效率
- 相比"脚本生成 + LLM审核"，节省 **~70% token**
- 质量显著提升
- 循环架构确保信息充分
