# 基于子Agent的自主学习系统设计 🤖

## 愿景
从"脚本驱动的学习"升级为"AI智能体管理的学习"

---

## 架构设计

```
┌─────────────────────────────────────────────────────────┐
│                    主Agent (乙维斯)                      │
│  - 学习目标设定                                           │
│  - 进度查询和管理                                         │
│  - 最终质量把关                                           │
└────────────────────┬────────────────────────────────────┘
                     │ 管理/对话/查询
                     ▼
┌─────────────────────────────────────────────────────────┐
│              学习子Agent (LearnerAgent)                  │
│  - 自主学习决策                                           │
│  - 学习进度管理                                           │
│  - 技能质量审核                                           │
│  - 定时上报进度                                           │
└────────────────────┬────────────────────────────────────┘
                     │ 执行学习任务
         ┌───────────┴───────────┐
         ▼                       ▼
┌─────────────────┐    ┌─────────────────┐
│  知识探索模块   │    │  技能生成模块   │
│  - GitHub       │    │  - 知识提取     │
│  - 技术博客     │    │  - 技能编写     │
│  - 官方文档     │    │  - 质量校验     │
└─────────────────┘    └─────────────────┘
```

---

## 核心特性

### 🧠 智能学习决策
- **LLM驱动的学习规划** - 用大模型决定学什么、怎么学
- **优先级排序** - 根据当前技能缺口自动排序学习主题
- **自适应调整** - 根据学习效果动态调整学习策略

### ✅ 智能质量审核
- **LLM质量评估** - 用AI评估技能质量，而不是简单的规则
- **结构化评审** - 多维度质量评分（实用性、完整性、准确性）
- **自动改进建议** - 对低质量技能给出具体改进建议

### 📊 智能进度管理
- **自然语言汇报** - 用对话方式汇报学习进度
- **定时上报** - 可配置的进度汇报频率
- **里程碑通知** - 重要学习成就自动通知

## 子Agent功能设计

### LearnerAgent 核心能力

#### 1. 学习规划
```
输入: 学习目标列表
输出: 今日学习计划
决策: 
  - 分析当前技能缺口
  - 评估学习资源可用性
  - 确定学习优先级
  - 分配学习时间
```

#### 2. 知识探索
```
任务: 浏览GitHub/技术博客
能力:
  - 使用agent-browser真实浏览
  - 智能筛选高质量内容
  - 提取关键知识点
  - 记录学习笔记
```

#### 3. 技能生成
```
输入: 学习笔记
输出: 完整技能(SKILL.md)
过程:
  - 知识结构化
  - 技能模板填充
  - 示例代码生成
  - 使用说明编写
```

#### 4. 质量审核
```
审核维度:
  - 实用性: 是否真的能用
  - 完整性: 覆盖是否全面
  - 准确性: 内容是否正确
  - 可读性: 表达是否清晰
  - 示例: 是否有实用示例

输出:
  - 质量评分(0-100)
  - 审核意见
  - 改进建议
  - 是否通过
```

#### 5. 进度汇报
```
汇报内容:
  - 已完成学习任务
  - 生成的技能列表
  - 质量审核结果
  - 遇到的问题
  - 下一步计划

汇报方式:
  - 定时汇报(如每2小时)
  - 里程碑汇报(完成重要目标)
  - 按需汇报(主Agent查询时)
```

## 技术实现方案

### 方案1: 使用 sessions_spawn (推荐)

**优势:**
- 利用OpenClaw原生的多Agent系统
- 自动处理会话隔离
- 完整的工具访问权限

**实现步骤:**

1. **创建LearnerAgent配置**
   - 在OpenClaw配置中添加learner agent
   - 配置专用的工作空间
   - 设置合适的模型(推荐zhipu/glm-5)

2. **创建LearnerAgent启动脚本**
   ```bash
   #!/bin/bash
   # start-learner-agent.sh
   
   # 使用sessions_spawn启动学习子Agent
   openclaw sessions spawn --agent learner --task "自主学习任务"
   ```

3. **创建学习任务定义**
   - 定义学习目标
   - 配置学习参数
   - 设置汇报频率

4. **实现进度汇报机制**
   - 使用sessions_send向主Agent汇报
   - 定时触发汇报
   - 里程碑事件触发

### 方案2: 使用 subagents 工具

**优势:**
- 更轻量级
- 可以动态steer子Agent
- 更好的控制能力

## 预期效果

### 相比脚本方式的提升

| 特性 | 脚本方式 | 子Agent方式 |
|------|---------|------------|
| 学习决策 | 固定规则 | LLM智能决策 |
| 质量审核 | 简单规则检查 | AI多维度评估 |
| 交互方式 | 命令行 | 自然语言对话 |
| 进度汇报 | 静态报告 | 动态对话汇报 |
| 自适应能力 | 无 | 智能调整策略 |
| 可扩展性 | 有限 | 高度可扩展 |

### 核心优势

1. **更智能** - 用AI做决策，而不是固定规则
2. **更自然** - 用对话方式管理学习
3. **更高效** - 智能优先级排序，学习效率更高
4. **更高质** - AI质量审核，技能质量更有保障
5. **更灵活** - 可以轻松扩展新的学习能力
