# 启动学习前的交互流程 🗣️

## 设计目标
在开始学习前，通过友好的对话明确用户需求，避免盲目学习！

---

## 🎯 交互问题设计

### 问题1: 学习深度（必选）

**问题**:
```
你想学习到什么程度？

A. 入门了解（快速上手，30-60分钟）
   - 适合: 想快速了解，不需要深入
   - 内容: 核心概念 + 1个简单示例
   - Token预估: 5K-10K

B. 系统学习（全面掌握，2-3小时）⭐推荐
   - 适合: 真正想掌握这个技能
   - 内容: 完整教程 + 多个示例 + 最佳实践
   - Token预估: 20K-40K

C. 深入精通（深度研究，4小时+）
   - 适合: 需要成为专家
   - 内容: 深度原理 + 高级技巧 + 实战案例
   - Token预估: 50K-100K

你的选择（A/B/C）：
```

**智能推荐**:
- 如果是新技术（如React 19）→ 推荐 B（系统学习）
- 如果是基础工具（如Git）→ 推荐 A（入门了解）
- 如果用户是开发者 → 推荐 B（系统学习）

---

### 问题2: 学习时间（必选）

**问题**:
```
⏰ 设置学习时间限制

推荐时长（根据学习深度）:
- 入门了解: 60分钟
- 系统学习: 120分钟 ⭐
- 深入精通: 240分钟

你想设置多少分钟？
- 直接输入数字（如: 90）
- 按回车使用推荐值
- 输入 0 表示不限制
```

**超时处理说明**:
```
💡 提示:
- 如果时间到了但没学完，我会通知你
- 询问是否需要继续
- 如果5分钟没有回复，自动保存并加入待学习列表
```

## 🤖 智能推荐逻辑

### 基于什么推荐？

**1. 技能缺口分析**
```
检查现有技能目录:
- 缺少 React 19 相关技能 → 推荐学习
- 已经有 Git 基础技能 → 不推荐或推荐进阶
```

**2. 用户历史分析**
```
看用户之前学过什么:
- 用户之前学过 React 18 → 推荐 React 19
- 用户是前端开发者 → 推荐 React, Tailwind
- 用户是后端开发者 → 推荐 Python, Docker
```

**3. 技术趋势分析**
```
GitHub Trending热门:
- Rust 持续热门 → 可以推荐
- WebAssembly 应用增多 → 可以推荐
```

**4. 当前时间分析**
```
如果是工作日晚上:
→ 推荐短时间学习（30-60分钟）

如果是周末:
→ 推荐系统学习（2-3小时）
```

## 💡 额外的友好功能

### 1. 快速跳过模式
```
如果用户说"直接开始，用推荐设置"
→ 跳过所有问题，直接用智能推荐开始
```

### 2. 保存偏好
```
记住用户的选择:
- 常用学习深度
- 常用时间限制
- 常用Token预算
下次自动推荐相同设置
```

### 3. 学习模式选择
```
提供两种模式:
- 交互模式（默认）- 问问题确认
- 快速模式 - 直接用推荐开始
```

---

## 🎯 总结

### 启动交互的好处
- ✅ 明确用户需求，避免盲目学习
- ✅ 合理设置时间和Token限制
- ✅ 用户有参与感，更满意
- ✅ 减少超支风险

### 交互原则
- 问题简洁，不要太多
- 提供清晰的选项
- 给出智能推荐
- 允许跳过/快速开始
- 记住用户偏好

---
