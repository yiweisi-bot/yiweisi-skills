
# 🧪 Agent Memory Ultimate 测试方案

**测试时间**: 2026-03-04  
**测试目标**: 验证记忆系统的核心功能  
**测试环境**: /root/.openclaw/workspace/skills/agent-memory-ultimate/

## 📋 测试清单

### 测试 1: 基础存储功能 ✅
**目标**: 验证能否成功存储新记忆

**步骤**:
1. 存储一条测试记忆
2. 验证记忆被保存
3. 检查数据库统计

**预期结果**:
- 记忆成功存储，获得 memory_id
- stats 显示活跃记忆数增加

---

### 测试 2: 关键词搜索功能 ✅
**目标**: 验证全文搜索功能正常

**步骤**:
1. 搜索已知关键词（如"甲维斯"、"GitHub"、"博客"）
2. 验证能找到相关记忆
3. 检查搜索结果相关性

**预期结果**:
- 能找到包含关键词的记忆
- 搜索结果按重要性排序

---

### 测试 3: 重要性分级功能 ✅
**目标**: 验证重要性参数生效

**步骤**:
1. 存储不同重要性的记忆（0.3, 0.7, 1.0）
2. 查询所有记忆并检查排序
3. 验证高重要性记忆排在前面

**预期结果**:
- 重要性 1.0 的记忆排在最前
- 重要性 0.3 的记忆排在最后

---

### 测试 4: 列出所有记忆功能 ✅
**目标**: 验证能正确列出所有记忆

**步骤**:
1. 使用 "*" 查询所有记忆
2. 检查返回的记忆数量
3. 验证每条记忆的元数据

**预期结果**:
- 返回所有活跃记忆
- 包含 id, type, importance, source 等信息

---

### 测试 5: 记忆来源追踪功能 ✅
**目标**: 验证 source 字段正确记录

**步骤**:
1. 检查现有记忆的 source 字段
2. 存储新记忆时指定 source
3. 验证 source 被正确保存

**预期结果**:
- 现有记忆显示 "memory_file:YYYY-MM-DD.md"
- 新记忆的 source 正确显示

---

### 测试 6: 统计信息功能 ✅
**目标**: 验证统计信息准确

**步骤**:
1. 运行 stats 命令
2. 验证活跃记忆数量
3. 验证软删除数量

**预期结果**:
- 活跃记忆数 = 当前实际记忆数
- 软删除数 = 0（初始状态）

---

### 测试 7: 内容完整性检查 ✅
**目标**: 验证记忆内容完整存储

**步骤**:
1. 存储一段较长的文本
2. 查询并验证内容完整性
3. 检查没有截断或损坏

**预期结果**:
- 记忆内容完整保存
- 没有意外截断

---

### 测试 8: 边界情况测试 ✅
**目标**: 验证系统在边界情况下的表现

**步骤**:
1. 存储空内容（应报错或拒绝）
2. 存储极长内容（10000+ 字符）
3. 存储特殊字符（中文、emoji、符号）

**预期结果**:
- 系统能正确处理各种输入
- 不会崩溃或数据损坏
